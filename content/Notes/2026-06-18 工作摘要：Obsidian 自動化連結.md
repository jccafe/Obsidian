# 2026-06-18 工作摘要：Obsidian 自動化連結

今天我們成功實作並部署了與 **Obsidian** 筆記軟體的自動化連結工具。以下為今日工作的詳細摘要：

---

## 🛠️ 完成的成果與工具

1. **自動化主腳本**：
   * 建立了 [obsidian_tool.py](file:///k:/Antigravity_folio/obsidian/obsidian_tool.py)，支援 `init`、`create`、`daily`、`open` 與 `test-api` 等核心命令。
2. **API 設定檔**：
   * 建立了 [config.json](file:///k:/Antigravity_folio/obsidian/config.json)，用於存放 Obsidian 的 Local REST API 金鑰與伺服器埠號。
3. **基礎資料夾結構**：
   * 自動初始化了 Obsidian 推薦的四個基礎目錄：`Notes/`、`Daily/`、`Templates/` 及 `Resources/`。

---

## 🔗 技術方案與連接機制

我們在實施計劃中討論並最終採用了 **方案二（Local REST API 模式）**，並完美融入了 **方案一（本地寫入模式）** 作為降級備份：

* **API 即時同步**：
  * 當 Obsidian 程式開啟時，工具會透過本地 HTTPS 協定 (`https://127.0.0.1:27124`) 直接與 Obsidian 進行即時通訊，寫入並更新筆記，甚至能控制 Obsidian 軟體自動開啟並聚焦至新建立的檔案。
* **智慧容錯處理**：
  * 我們在腳本中加入了自動過濾機制，會自動剝離 API Key 設定檔中不小心帶入的 `Bearer ` 前綴，避免了 `401 Unauthorized` 錯誤。
* **離線降級機制**：
  * 當偵測到 API 金鑰未設定，或者 Obsidian 程式關閉（無法建立 HTTP 連線）時，工具會自動警告並**降級為本地檔案寫入**。程式直接對硬碟檔案進行讀寫，保證自動化工作不中斷。
